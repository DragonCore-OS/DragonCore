## Why Dragon / 为何是龙

### The Chinese Word for Lobster / 中文里的"龙虾"

In English, "lobster" is just lobster.  
But in Chinese, the word for lobster is **龙虾**:

- **龙** (lóng) = dragon  
- **虾** (xiā) = shrimp

This creates a cultural contrast that **does not exist in English**.

In Chinese, 龙虾 does not evoke the majesty of a dragon.  
It is, quite literally, a **shrimp with a dragon-like name**.

### The Metaphor / 隐喻

Many AI systems today are like **lobsters (龙虾)**:
- They borrow the dragon's name but remain shrimp in essence
- Scattered, sideways-moving (横向移动)
- Hard shells (硬壳), soft governance
- Grasping at tasks without coordination
- Disposable tools pretending to be institutions

**DragonCore is the True Dragon (真龙):**
- **Order** — 三省九部，分层治理
- **Legitimacy** — traceable authority, formal arbitration  
- **Coordination** — deliberation, not chaos
- **Continuity** — archive, inheritance, civilizational memory

**真龙，不是龙虾。**  
*True Dragon. Not a shrimp with a borrowed name.*

## How It Works: A Governance Run

### Step 1: Submission / 提交
An **engineering agent** submits a proposal (e.g., "Refactor the authentication module") to the **Secretariat (中书省)**.

### Step 2: Strategy Review / 策略审查
**Tianji (天玑)** evaluates: Does this align with our architecture? Is this the right time?  
**Zhugeliang (诸葛亮)** analyzes: What are the long-term implications? Strategic risks?

### Step 3: Risk Assessment / 风险评估
**Tianxuan (天璇)** identifies technical risks.  
**Baihu (白虎)** attacks the proposal: "What if this breaks authentication for 10,000 users?"  
If risks are unacceptable, **veto** — back to engineering.

### Step 4: Implementation Review / 实现审查
**Luban (鲁班)** reviews code quality: Is this craftsmanship or a hack?  
**Kaiyang (开阳)** checks technical feasibility: Can this actually be built?

### Step 5: Quality Gate / 质量门禁
**Yuheng (玉衡)** runs tests, checks coverage, reviews documentation.  
Strict standard: No approval without 85%+ test coverage.

### Step 6: Resource Allocation / 资源分配
**Xiwangmu (西王母)** assigns compute resources.  
**Tianquan (天权)** coordinates which agents work on what.

### Step 7: Execution / 执行
**Nezha (哪吒)** rapidly deploys.  
**Engineering Ministry (夏官)** implements.

### Step 8: Verification / 验证
**Yangjian (杨戬)** checks: Is the reported completion real?  
Uses "third eye" to detect fake progress.

### Step 9: Audit / 审计
**Baozheng (包拯)** audits: Any rule violations? Any corruption?  
**Zhongkui (钟馗)** purges any toxic elements discovered.

### Step 10: Final Approval / 终局裁决
**Tianshu (天枢)** gives final approval.  
Decision is **binding**.

### Step 11: Archive / 归档
**Yaoguang (瑶光)** indexes all artifacts.  
**Fengdudadi (丰都大帝)** ensures proper closure.  
Decision log preserved for eternity.

## Why 19? / 为什么是十九席？

The number **19** is deliberate:

- **7** (Big Dipper) — Core governance, the "seven stars" that guide
- **4** (Four Symbols) — Cardinal oversight, the four directions
- **8** (Mythological) — Specialized functions, the eight trigrams of the I Ching

**7 + 4 + 8 = 19**

In Chinese numerology:
- **1** represents unity and origin (the Tao)
- **9** represents completion and permanence (the highest yang number)

**19** combines origin (1) with completion (9) — a system that is both **foundational** and **complete**.
